Google Cloud Platform (GCP) organizes its resources in a hierarchical structure to help manage access control, billing, and policies efficiently.

üîù 1. Organization (Root Node)

  • Top-most node in the hierarchy.
  • Represents your entire company or domain, e.g. example.com.

  • Optional layer to group projects logically.

üì¶ 3. Projects

  • The core unit of resource organization and management in GCP.
  • Every GCP resource (VM, storage bucket, etc.) must belong to a single project.
  • Each project has:
    • Project ID (unique globally)
    • Project Name
    • Project Number
  • Billing, APIs, and IAM roles are managed at the project level.

‚öôÔ∏è 4. Resources

  • These are the actual GCP services you create and use ‚Äî like Compute Engine VMs, Cloud Storage buckets, BigQuery datasets, etc.

üîê IAM Policy Inheritance

  • IAM (Identity and Access Management) policies cascade down the hierarchy.
  • A role granted at the Organization level applies to all folders, projects, and resources beneath it.
  • Lower levels can override or add more specific roles.

Note: This hierarchical structure enables centralized management while allowing delegation and fine-grained control at different levels of your organization.
